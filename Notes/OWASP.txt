Injection

-user supplied data
-queries or non-parameterized calls
-any commands

-always validate user input both front and backend
-keep queries away from special characters
-LIMIT and other SQL controls

https://owasp.org/www-project-top-ten/2017/A1_2017-Injection

Broken Authentication

-cedential stuffing 
-bad passwords
-T&E
-no multi-factor authentication
-session ID

-no week passwords
-no default credentials
-modern evidence based password policies
-failed attempt limits
-session ID not in URL

https://owasp.org/www-project-top-ten/2017/A2_2017-Broken_Authentication
